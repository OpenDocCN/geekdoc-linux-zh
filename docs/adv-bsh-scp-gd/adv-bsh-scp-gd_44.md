# 38.1\. 作者注释

> 原文：[`tldp.org/LDP/abs/html/authorsnote.html`](https://tldp.org/LDP/abs/html/authorsnote.html)

|   |  ***教人学习**

*(教人学习，你自己也会学到。)** |

我是如何开始写脚本编程书的？这是一个奇怪的故事。似乎几年前我需要学习 shell 脚本编程——还有什么比读一本关于这个主题的好书更好的方法呢？我正在寻找一本涵盖该主题所有方面的教程和参考书。我正在寻找一本能够将复杂的概念彻底解释清楚，并以详尽的细节、带有良好注释的例子进行解释的书。1 实际上，我正在寻找的就是这本书，或者非常类似的书。不幸的是，它并不存在，如果我想得到它，我就必须自己写。所以，我们就到了这里，朋友们。

这让我想起了关于一个疯狂教授的轶事。这家伙疯狂得像只疯鹅。一看到书，无论是什么书——在图书馆、书店，还是任何地方——他都会完全痴迷于自己能写这本书，应该写这本书——并且写得更好。于是他会急忙回家，着手写一本具有相同标题的书。几年后他去世时，据说他拥有几千本书，可能连阿西莫夫都感到羞愧。这些书可能并不怎么样，谁知道呢，但这真的重要吗？这是一个实现了自己梦想的人，即使他为此着迷，被它驱使……而且我忍不住要钦佩这位老先生。
